# Migration Guide — v0.2 → v0.3

이 가이드는 설계 문서 v0.2 기준으로 작성된 외부 도구/설정을 v0.3 으로 이주하는 방법을 다룹니다.

## 주요 변경 사항

### ADR-006 — `enabled` + `mode` → 단일 `state`

**Before (v0.2)**:
```json
{
  "enabled": true,
  "mode": "active",
  ...
}
```

**After (v0.3)**:
```json
{
  "schemaVersion": "1.1.0",
  "state": "active",
  ...
}
```

### ADR-007 — Goal 인터페이스 공식 정의

Goal 타입이 공식 스키마로 고정됨. ID는 `goal-` 접두사 필수.

**Before (v0.2)**:
```ts
// 구현체마다 다른 Goal 모양
type Goal = { id: string; description: string; ... };
```

**After (v0.3)**:
```ts
import type { Goal } from '@agent-platform/core';
// Goal.id 는 'goal-{uuid}' 패턴
// status: 'active' | 'paused' | 'completed' | 'abandoned'
// progress: [0, 1]
// createdBy: 'user' | 'ego'
```

## 자동 마이그레이션

CLI 가 대부분을 처리합니다:

```bash
# 기존 ~/.agent/ego.json 이 v0.2 형식일 때
agent ego status
# → 아직 로드 가능 (관대한 파싱)

# 명시적 마이그레이션
agent ego active   # 자동으로 enabled/mode 제거 + state 설정
```

`agent ego <action>` 은:
1. 기존 파일 읽기
2. `delete config.enabled; delete config.mode;`
3. `config.state = action;`
4. `config.schemaVersion = '1.1.0';`
5. 저장

## 수동 마이그레이션 (코드)

코드에서 직접 `EgoFullConfig` 를 다루는 경우:

### 매핑 규칙

| v0.2 | → | v0.3 |
|------|---|------|
| `enabled: false` | → | `state: 'off'` |
| `enabled: true, mode: 'passive'` | → | `state: 'passive'` |
| `enabled: true, mode: 'active'` | → | `state: 'active'` |

### 마이그레이션 함수

```ts
interface OldEgoConfig {
  enabled: boolean;
  mode: 'passive' | 'active';
  // ... 나머지
}

interface NewEgoConfig {
  schemaVersion: string;
  state: 'off' | 'passive' | 'active';
  // ... 나머지 (거의 동일)
}

function migrate(old: OldEgoConfig): NewEgoConfig {
  const { enabled, mode, ...rest } = old as unknown as Record<string, unknown>;
  const state = !enabled ? 'off' : (mode ?? 'active');
  return {
    schemaVersion: '1.1.0',
    state: state as 'off' | 'passive' | 'active',
    ...rest,
  } as NewEgoConfig;
}
```

## 신규 필수 필드

v0.3 `ego.json` 은 다음 필드들이 **새로 추가/의무화** 되었습니다:

### `thresholds.maxCostUsdPerDay`

일일 비용 한도. 초과 시 `state` 가 자동으로 한 단계 다운그레이드됩니다.

```json
"thresholds": {
  ...
  "maxCostUsdPerDay": 5.0      // 신규
}
```

### `fastPath.targetRatio` + `measurementWindowDays`

Fast-path 비율 실측 목표. Phase EGO-4 에서 실측 대비 경보/튜닝.

```json
"fastPath": {
  ...
  "targetRatio": 0.75,               // 신규
  "measurementWindowDays": 7         // 신규
}
```

### `memory.onTimeout`

메모리 검색 타임아웃 시 동작. v0.2 는 정의되지 않았음 (구현마다 달랐음).

```json
"memory": {
  ...
  "onTimeout": "empty_result"   // "empty_result" | "cached" | "abort"
}
```

### `persona` 섹션

페르소나 시스템 설정. v0.2 에는 없었음.

```json
"persona": {
  "enabled": true,
  "storePath": "~/.agent/ego/persona.json",
  "snapshot": {
    "maxTokens": 250,
    "topRelevantBehaviors": 3,
    "topRelevantExpertise": 3,
    "includeRelationshipContext": true
  }
}
```

### `errorHandling` 섹션

EGO LLM 실패 처리 규칙. v0.2 는 구현체마다 달랐음.

```json
"errorHandling": {
  "onLlmInvalidJson": "passthrough",
  "onLlmTimeout": "passthrough",
  "onLlmOutOfRange": "passthrough",
  "onConsecutiveFailures": {
    "threshold": 5,
    "action": "disable_llm_path",
    "cooldownMinutes": 15
  }
}
```

### `audit.retentionDays`

감사 로그 보존 기간.

```json
"audit": {
  ...
  "retentionDays": 90           // 신규
}
```

### `goals.autoDetectCompletion`

Goal 자동 완료 감지. v0.2 에서는 참조만 되고 정의되지 않았음.

```json
"goals": {
  ...
  "autoDetectCompletion": true   // 신규
}
```

## 코드 API 변경

### `EgoDecision` — 변화 없음

4가지 action (`passthrough`/`enrich`/`redirect`/`direct_response`) 과 각 payload 모양은 **동일하게
유지**. 이미 v0.2 에서 이 이름들이 안정적이었음.

### `EgoMetadata` 에 `egoDecisionId` 추가

```ts
interface EgoMetadata {
  egoDecisionId: string;        // 신규 — ID 추적
  decisionReason: string;
  confidenceScore: number;
  decisionTimeMs: number;
  llmCostUsd?: number;          // 신규
  goalsUpdated: Goal[];
}
```

이 ID 는 enriched message 의 `channel.metadata._egoDecisionId` 로 전파되어 사용자 피드백과 감사
로그를 역추적 가능하게 합니다.

### `Session.status` 확장

`'redirected'` 추가. v0.2 에는 `'active'|'hibernated'|'archived'` 만.

```ts
type SessionStatus = 'active' | 'hibernated' | 'archived' | 'redirected';
```

### `SessionManager.updateSession` 추가

Contract 신규 메서드:
```ts
interface SessionManager {
  // ... 기존 ...
  updateSession(sessionId: string, patch: SessionPatch): Promise<Session>;  // 신규
}
```

## 감사 로그 태그 확장

v0.3 은 20+ 태그를 정의. 주요 신규:

- `ego_state_transition`
- `ego_redirect`
- `daily_cost_cap_hit`
- `ego_circuit_open`
- `llm_schema_mismatch`, `llm_out_of_range`, `llm_inconsistent_action`, `llm_invalid_target`
- `memory_timeout`
- `persona_snapshot_truncated`, `persona_field_saturated`

v0.2 감사 로그를 조회하는 코드는 새 태그를 이해하도록 업데이트 필요.

## 페르소나 (ego-persona.md v0.2)

v0.1 과 거의 호환. 주요 차이:

### 진화 규칙 신규 — `confirmThreshold` + `confirmWindowCount`

상반 방향 변화 감지용. v0.1 에서는 정의되지 않았음.

### Maturity 전이는 upgrade-only

v0.1 에서는 `totalInteractions` 가 낮아지면 (이상한 재설정 등) maturity 가 downgrade 될 수 있었으나,
v0.2 에서는 upgrade-only. 즉 `established` → `developing` 은 절대 일어나지 않음.

## 체크리스트

이주 작업 시:

- [ ] `~/.agent/ego/ego.json` 백업
- [ ] `agent ego <state>` 실행으로 자동 마이그레이션
- [ ] 새 필수 필드 확인 (`maxCostUsdPerDay`, `memory.onTimeout`, `persona`, `errorHandling`)
- [ ] 코드에서 `EgoFullConfig` 타입을 쓰는 곳 빌드 재실행 (컴파일러가 누락 필드 알림)
- [ ] 감사 로그 조회 쿼리에 새 태그 포함
- [ ] Goal 저장 위치 `~/.agent/ego/goals.json` 구조 확인 (신규 파일이면 자동 생성)
- [ ] `PersonaManager` 쓰는 코드에서 `EvolutionRules.confirmThreshold` 등 신규 필드 기본값 확인
